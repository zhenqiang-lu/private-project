1. 网络连接

1.1 网络连接

网络摄像机与电脑之间常用连接方式主要有两种
在通过网络访问网络摄像机之前，首先需要获取它的IP地址，用户可以通过“GW Device Manager”软件来搜索网络摄机的IP地址。
安装随机光盘里面的“GW Device
Manager”软件，单击“搜索”。软件会显示当前局域网内中正在运行的网络摄像机的IP地址、端口号、MAC地址、S/N、UID、版本信息、型号等。
若搜索出来的IP地址和电脑IP地址不在同一网段，请通过“GW Device Manager”软件修改网络摄像机IP地址、子网掩码等参数。
在“GW Device Manager”软件中，选择要修改的摄像机，右击选择“网络配置”，然后输入新的IP地址、子网掩码等参数。

说明:

IPC IP地址出厂默认是开启DHCP，建议把IPC接入有DHCP Server的网络中，如果网络中没有DHCP Server，IP地址则会变成192.168.0.66。

第一次使用WEB客户端时，需要加载“VLC media player”，具体操作步骤如下:

步骤2: 连接成功后，选择合适的语言，输入用户名和密码。出厂默认管理员用户名为: admin,密码为: admin
步骤3 如何运行WEB插件:
•打开浏览器输入IPC IP地址
•您会看到一个运行加载项提示，请按照提示进行
通过浏览器客户端您可以:
•实时查看图像
•录像回放 Record
•保存录像到本地电脑和设置IPC参数
•您无需将IPC接入监视器，只需将IPC接入本地电脑中即可
•快速简单设置IPC参数，更改设置，录像质量和时间。
电脑最低配置要求：
2.0GHZ或以上CPU
2GB 或以上RAM
10/100Mbps网络
1024x768或以上分辨率
支持电脑系统如下：
Microsoft Windows XP, Microsoft Windows Vista, Microsoft Windows 7,Microsoft Windows 8,Microsoft Windows 10 Mac OS

本文所涉及到的浏览器均为IE浏览器，请使用IE 9或以上浏览器。

4.1.1 Audio Video

步骤1 选择“Settings > Config Media > Audio Video”
步骤2 “Audio Video”参数具体设置请参见如下:
Standard:根据需要设置PAL或NTSC
PAL：1-25帧/秒或1-50帧/秒
NTSC：1-30帧/秒或1-60帧/秒
Stream type:码流类型包括主码流和子码流。
主码流：用于摄像机录像及显示。高质量的视频流
子码流：摄像机将通过网络发送视频流至远程摄像机，此为低质量视频流，视频文件大小会缩减，便于网络传送
Frame Rate: 表示摄像机1秒内录像多少幅图片。减少帧率将不会节省硬件存储空间，但是会提高每一帧的数据数率。帧率会随分辨率的不同而变化
Codec:可选择H264或H265
H264：能够在低码率情况下提供高质量的视频图像，可以工作在实时通信低延时模式下
H265 :在同样高质量的视频图像下，能够节省50%的存储空间
Bitrate Mode:编码模式包括固定码流和变码流
固定码流：在任何场景中，码流大小不变化
变码流：在复杂和有移动物体的场景中，码流会增大，反之会减少，便于节省存储空间
Resolution :表示有多少个像素点组成图像。值越大，像素点就越多，图像就越清晰
Video Quality:图像质量包括“最好，好，较好，一般，较差”等五个选项
Video Rate: 表示摄像机用于录像的实际数据量。比特率越高，录像占用存储空间就越大。简言之，录像编码比特率越高，图像效果越好
I Frame Interval: 表示两个I帧之前的P帧的数量，数值越大，关键帧越少，反之，关键帧越多。增加关键帧个数可改善质量，但是同时增加带宽和网络负载。最大为100，建议设置为帧率的2倍，默认值为５０
Audio Enable:勾选此项表示开启音频输入输出，反之则关闭音频输入输出
Codec:选择音频编解码格式G.711U /G.77A，建议选择G.711A
步骤3 单击“保存”，完成视频配置

4.1.3 Image

•Hue: 调节色彩明暗程度。数值0-100可选，默认值为50
•Brightness: 调节图像的整体亮度。值越大图像越亮，值越小图像越暗。值设的过大时，图像容易发朦。数值0-100可选，默认值为为50
•Contrast: 调节图像的对比度。值越大图像明亮反差越大，反之越小。值设的过大时，图像暗的地方太暗，亮的地方容易过曝。数值0-100可选，默认值为50
•Saturation: 调整颜色深浅。值越大彩色将越浓，反之相反。数值0-100可选，默认值为50
•Sharpness: 调节图像的边缘的锐利程度，值越大边缘越明显，反这相反，设得较大时，图像容易产生噪点。数值0-100可选，默认值为50
•Mirror: 调节图像水平镜像，左右可选。默认值为左
•Turn: 调节图像垂直翻转：上下可选。默认值为上
•Camera Angle: 该功能可改变图像的方向。正常、90°、180°、270° 可选。默认值为正常
•3D DNR: 该功能可能会降低图像的噪点，值越大，效果越好，但是图像会朦，有拖尾的现象。关闭、开启可选。默认值为关
•LUX: 该功能可设置CDS照度值，值越大LED越容易开启。当为0时，LED不开。当为9时，LED常开。LUX：0-9可选
•Dewarping: 该功能可矫正鱼眼图像。关闭、开启可选。默认值为关闭
•Led intensity: 该功能可设置LED强度，值越大电流越大。当为0时，LED关闭。LED强度：0-9可选
•Exposure Mode:  自动：图像的整体亮度在正常曝光范围内根据不同场景亮度自动调整。
 手动：可手动选曝光值。
•曝光：自动、手动可选。默认值为自动
•shutter: 该功能可调节快门速度，在固定光圈下，值越大图像越亮，但是会过曝。值越小图像越暗，快门调节范围：1/25-1/100000。默认值为不可用
•BLC Mode:
WDR：该功能可以将高亮区亮度降低，低亮度区域亮度提高，尽量使高亮和低亮区域景物都能看清。数值0-100可选，默认值为为50
 DWDR：该功能可以将低亮区域亮度提高，高亮区域亮度不变。值越大时，会产生发朦现象。数值0-100可选，默认值为为50
 BLC：根据所处场景自动进行曝光，以看清图像中最暗的区域为目标。数值0-100可选，默认值为为50
  HLC：该功能会抑制高亮区域的亮度，减少光晕区域的大小，整个画面亮度降低。数值0-100可选，默认值为为50
•背光补偿：关闭、WDR、DWDR、背光补偿、强光仰制。默认值为关闭
•FLC :
50Hz：在50Hz的情况下，根据场景亮度，自动调整曝光，确保图像不出现横条纹。
 60Hz: 在60Hz的情况下，根据场景亮度，自动调整曝光，确保图像不出现横条纹。
 室外：该模式下可切换曝光模光，达到相应曝光模式下的效果
•防闪烁：室外、50Hz、60Hz可选。默认值为室外
•Day/Night:
彩色：只显示彩色图像
 黑白：只显示黑白图像
 自动：根据CDS照度情况来切换显示彩色或黑白图像
 时间：根据设定的时间来切换彩色或黑白图像
•Day/Night：彩色、黑白、自动、时间可选。非红灯机默认值为彩色，红灯机默认值为自动

•DHCP 根据需要设置启用或不启用。Enable or Disable DHCP
 启用DHCP: 一个连接你网络的摄像机（通常是一个路由器）会自动分配IP地址给网络连接的装置。  Enable DHCP: DHCP is a system where one device on your network (usually a router) will automatically assign IP address to device connected to the network.  不启用DHCP: 静态网络要求所有的摄像机都有他们手动定义的IP地址，因为没有专门的摄像机来自动分配IP地址。  Disable DHCP(Static): Static networks require all devices to have their IP addresses manually defined, as there is no device dedicated to automatically assign IP address.
•IPv4/IPv6 Address 正如家庭和办公室需要一个地址来确定它们在网络中的位置。
 Just as Home and Office needs to have an address which identifies their location on the road network. The camera uses IPv4 addressing, which consists of four groups of numbers between 0 and 255, separated by periods. For example, a typical IP address might be “192.168.1.37”or similar.  IPv4地址:由0和255之间的四组数字组成，并由句点分隔。例如，一个典型的IP地址可能是“192.168.1.24”或与之类似的。  IPv6地址:IPv6的地址长度为128位，是IPv4地址长度的4倍，采用十六进制表示，并由冒号分隔。例如，一个典型的IP地址可以是“2001:250:3000:1::1:7”或与之类似的  The length of the IPv6 address is 128 bits, which is four times the length of the IPv4 address, expressed in hexadecimal and separated by colons. For example, a typical IP address can be "2001: 250: 3000: 1 :: 1: 7 "or similar. •IPv4 Subnet  If the IP address is like a street address, then a sub-network is like your neighborhood.
 This will be formatted in a similar way to the IP address (ie. four numbers up to 255 separated by periods) but contain very different numbers.  In the above example, the Subnet might be something like:“255.255.255.0”. •Gateway  This is the address of the “way to the Internet.” To continue the road analogy, this is like your local access point to the highway. This is an IP address in the same format as the others, and is typically very similar to the IP address of the Camera.  To continue the above example, it might be something such as: “192.168.1.1”.

Appendix 1 Glossary of Terms

1 WDR

当在强光源照射下的高亮度区域及阴影、逆光等相对亮度较低的区域在图像中同时存在时，摄像机输出的图像会出现明亮区域因曝光过度成为白色，而黑暗区域因曝光不足成为黑色，严重影响图像质量。宽动态这一技术是同一时间曝光两次，对明亮部分进行最合适的快门速度曝光，然后再对暗的部分用最合适的快门速度曝光，然后将两个图像进行DSP处理重新组合，使明亮的部分和暗的部分可以看得清楚。



2 DWDR

通过层次补偿来提高摄像主体的对比度，图像的层次感，在低亮度和高亮度区域，经过DWDR功能提供最佳的图像灰度等级补偿，提高了整个画面的可视性及明度。



3 BLC

BLC，也称为逆光补偿，是把画面分成几个不同的区域，每个区域分别曝光。使监视器上的主体画面明朗，大大降低背景画面与主体画面的主观亮度差，虽然改善了拍摄主体的亮度，整个视场的可视性得到改善.但是图像质量或多或少会劣化下降。



4 HLC

HLC的功能就是采用DSP技术。有效抑制强光照射造成的光晕偏大，视频图像模糊，开启该功能后能自动分辨强光点，对强光进行抑制，然后再对强光点附近区域进行补偿以获得较清晰的图像。



5 3DNR

3D降噪（3D DNR）监控摄像机通过对前后两帧的图像进行对比筛选处理，从而将噪点位置找出，对其进行增益控制，从而减少噪点。采用3D降噪的摄像机，图像噪点会明显减少，但是图像会比较蒙，出现有拖尾的情况。



6 IPv6

IPv6是IETF（互联网工程任务组，Internet Engineering Task Force）设计的用于替代现行版本IP协议（IPv4）的下一代IP协议。由于IPv4最大的问题在于网络地址资源有限，严重制约了互联网的应用和发展。IPv6的使用，不仅能解决网络地址资源数量的问题，而且也解决了多种接入设备连入互联网的障碍。



7 FTP

FTP 是文件传输协议，用于Internet上的控制文件的双向传输。同时，它也是一个应用程序。基于不同的操作系统有不同的FTP应用程序，而所有这些应用程序都遵守同一种协议以传输文件。在FTP的使用当中，有两个概念："下载"（Download）和"上传"（Upload）。"下载"文件就是从远程主机拷贝文件至自己的计算机上；"上传"文件就是将文件从自己的计算机中拷贝至远程主机上。用Internet语言来说，用户可通过客户机程序向（从）远程主机上传（下载）文件。



8 E-mail

E-mail是—种用电子手段提供信息交换的通信方式，是互联网应用最广的服务。通过网络的电子邮件系统，用户可以以非常低廉的价格（不管发送到哪里，都只需负担网费）、非常快速的方式（几秒钟之内可以发送到世界上任何指定的目的地），与世界上任何一个角落的网络用户联系。



9 DHCP

DHCP是一个局域网的网络协议，使用UDP协议工作，通常被应用在大型的局域网络环境中，主要作用是集中的管理、分配IP地址，使网络环境中的主机动态的获得IP地址、Gateway地址、DNS服务器地址等信息，并能够提升地址的使用率。DHCP协议采用客户端/服务器模型，主机地址的动态分配任务由网络主机驱动。当DHCP服务器接收到来自网络主机申请地址的信息时，才会向网络主机发送相关的地址配置等信息，以实现网络主机地址信息的动态配置。



10 DDNS

DDNS是将用户的动态IP地址映射到一个固定的域名解析服务上，用户每次连接网络的时候客户端程序就会通过信息传递把该主机的动态IP地址传送给位于服务商主机上的服务器程序，服务器程序负责提供DNS服务并实现动态域名解析。也就是说DDNS捕获每次变化的IP地址，然后将其与域名相对应，这样其他上网用户就可以通过域名来进行交流。而最终客户所要记忆的全部，就是记住动态域名商给予的域名即可，而不用去管他们是如何实现的。



11 PPPOE

PPPOE是一种先进的协议（点到点连接协议），使摄像机通过调制解调器直接连接。它利用以太网将大量主机组成网络，通过一个远端接入设备连入因特网，并对接入的每一个主机实现控制、计费等功能。



12 RTSP

RTSP是实时流传输协议，是TCP/IP协议体系中的一个应用层协议。该协议定义了一对多应用程序如何有效地通过IP网络传送多媒体数据。用来控制声音或影像的多媒体串流协议，并允许同时多个串流需求控制。



13 ONVIF

ONVIF标准为网络视频设备之间的信息交换定义通用协议，包括装置搜寻、实时视频、音频、元数据和控制信息等。确保不同厂商生产的网络视频产品具有互通性。



14 H265

H.265是H.264之后所制定的新的视频编码标准。H.265标准围绕着现有的视频编码标准H.264，保留原来的某些技术，同时对一些相关的技术加以改进。新技术使用先进的技术用以改善码流、编码质量、延时和算法复杂度之间的关系，达到最优化设置。相同的图像质量H265比H264要节省一半的码流，从而减少网络占用率和存储空间。



15 HTTPS

HTTP协议被用于在Web浏览器和网站服务器之间传递信息。HTTP协议以明文方式发送内容，不提供任何方式的数据加密，如果攻击者截取了Web浏览器和网站服务器之间的传输报文，就可以直接读懂其中的信息。为了解决HTTP协议的这一缺陷，需要使用另一种协议：HTTPS。为了数据传输的安全，HTTPS在HTTP的基础上加入了SSL协议，SSL依靠证书来验证服务器的身份，并为浏览器和服务器之间的通信加密。
